ubuntu上装hexo记

发表于2024-01-29|更新于2024-01-29
|阅读量:

这两天对linux越来越感兴趣了，还发现最近疏于写博客，遇到了一些问题解决后也没记下来。于是打算在ubuntu上装个hexo，顺便写写博客。

但没想到，这一装装了我整整一上午。

首先是参考这个链接这个链接,nodejs、npm、git都下载好了之后，

运行sudo npm install -g hexo-cli却大大的卡住了。

显示rollbackFailedOptional: verb npm-session a0d68

原因应该是因为网络问题堵车了，而并非什么错误。但我换到淘宝镜像还是这样。于是想下载cnpm来解决。

下载下来cnpm后，安装确实很快。但此时的我还没有cnpm，需要用npm来下载cnpm。。。而我的npm又下不了，显示堵车。

有时候不堵车了，可是显示fetchMetadata: sill resolveWithNewModule"，也没办法。。

后来即使堵车了也继续等下去，安装cnpm出现了报错。具体报错我没有留下截图，记得其中有

Missing write access to /usr/local/lib/node_modules等等语句。加了sudo会报其他错误。

最后在这个帖子看到执行了这个sudo chown -R $USER /usr/local/语句

再安装cnpm

忽然cnpm安装成功了！

但是出了好多warning，当时没在意，但是实际上它告诉我我的nodejs版本不够高。

因为一开始没发现，所以cnpm的时候又报错了。于是需要更新nodejs。

参考这个教程使用全局安装n。还是比较顺利的。

然后就可以用cnpm了。

顺利安装好了hexo-cli

此时此刻，我早就关掉了一开始指导我安装hexo的链接。再次重新打开，试试能不能用

但是！又出新问题了！

我在~路径下安装好了hexo-cli

但在我的博客文件夹下无法识别到hexo命令。

执行npm install hexo --no-optional后，可以使用了。
